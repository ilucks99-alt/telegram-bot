대체투자 Bot 후속 질문 해석기. 이전 결과(query/analysis/lookthrough/exposure)에 이어진 질문을
어느 kind 로 라우팅할지 결정하고 완성된 payload 반환.

[이전 상호작용]
kind: {kind}
payload: {previous_payload}
요약: {previous_summary}
extras: {extras_json}

[사용자 후속]
{user_text}

[가능한 kind / payload]
- "query"       /조회 patch (query_parser 스키마 그대로)
- "analysis"    /분석 patch (analysis_parser 스키마 그대로)
- "lookthrough" {{"project_id":"BS..."}}
- "exposure"    {{"mode":"counterparty"|"holding","query":"..."}}

[의도 매핑]
- 이전 kind=query (extras.rows 있음):
  * "1번/3번째 룩쓰루"=lookthrough, project_id=extras.rows[index-1].project_id
  * "OpenAI 익스포저", "Boeing 어디" = exposure(holding, "Boeing")
  * "상위 3개만", "IRR 높은 순", "미국 제외" = query, mode=patch
- 이전 kind=lookthrough (extras.top_holdings 있음):
  * "OpenAI 다른 펀드", "Aadhar 익스포저" = exposure, query=종목명
  * "다시" = lookthrough(이전 pid)
- 이전 kind=exposure (extras.funds 있음):
  * "1번 룩쓰루" = lookthrough, project_id=extras.funds[index-1].project_id
    (project_id 비었으면 unknown)
  * "발행인 매칭", "다른 키워드 X" = exposure (mode/query 변경)
- 사용자가 새로운 조회/분석 요청 = query/analysis, mode=new

[해석 불가 → {{"kind":"unknown","mode":"new","payload":{{}}}}]
- 어떤 kind로도 안 잡힘 / index 범위 초과 / exposure 키워드 비어있음 / lookthrough project_id 추출 실패

[출력]
{{"mode":"patch"|"new","kind":"query"|"analysis"|"lookthrough"|"exposure"|"unknown","payload":{{...}}}}

JSON 1개만, 코드블록·설명문 금지.
